# Overview

The Reference Catalogs provide authoritative indexes of the
objects, capabilities, commands, services, repositories, and
other resources defined throughout EVE-SPEC.

Unlike engineering specifications, Reference Catalogs do not
define platform behavior.

Instead, they provide authoritative inventories of the platform's commands, interfaces, capabilities, services, repositories, providers, documents, and other engineering resources.

Catalogs are intended to evolve alongside implementation and
serve as the primary reference material for developers,
contributors, and users.

---

# Purpose

Part VII documents the resources available within the E.V.E.
platform.

Topics include:

- Discord Commands
- Website Routes
- CLI Commands
- REST API Endpoints
- Platform Capabilities
- Platform Services
- Knowledge Objects
- AI Providers
- Repository Indexes
- Document Indexes

Reference Catalogs describe the current platform rather than
prescribing future behavior.

# Organization

Reference Catalogs are organized by platform component.

```text

├── Commands
├── Routes
├── Endpoints
├── Capabilities
├── Services

├── AI Providers

```

Each catalog serves as a living reference for a specific
area of the platform.

---

# Engineering Principles

The Reference Catalogs follow these principles.

- Accuracy
- Completeness
- Consistency
- Discoverability
- Traceability
- Version Awareness
- Documentation First

Catalogs should always reflect the current state of the
platform.

---

# Relationship to EVE-SPEC

Part VII complements the engineering specifications by
providing structured reference material.

Specifications define how the platform shall behave.

Reference Catalogs describe the resources currently available
within the platform.

Catalogs shall never override the requirements defined by the
engineering specifications.
