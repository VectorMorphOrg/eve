# Purpose

This document provides the authoritative catalog of
engineering documents recognized by the Evolutionary Virtual
Engineer (E.V.E.).

Unlike engineering specifications, this catalog does not
define document behavior.

Instead, it serves as the centralized inventory of document
types managed, indexed, and referenced throughout the
E.V.E. platform.

Document lifecycle and behavior are defined by the
corresponding engineering specifications.

---

# Scope

This catalog includes:

- Engineering Specifications
- Reference Catalogs
- Architecture Decision Records
- Implementation Guides
- Developer Guides
- Repository Documents
- Governance Documents
- Community Documents

Additional document types may be introduced as the platform
evolves.

# Document Naming Guidelines

Documents should:

- Represent one primary subject.
- Have a clearly defined purpose.
- Be uniquely identifiable.
- Follow EVE-SPEC naming conventions.
- Be version controlled.
- Remain discoverable through the Knowledge Layer.

Documents should serve as authoritative sources of
engineering knowledge.

---

# Relationship to EVE-SPEC

This catalog complements the Core Platform Specifications,
Knowledge Layer Specification, and Developer Guides.

Engineering specifications define document behavior.

This document serves only as the authoritative inventory of
document types recognized by the E.V.E. platform.
