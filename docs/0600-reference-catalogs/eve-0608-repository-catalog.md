---

# Purpose

This document provides the authoritative catalog of
repositories recognized by the Evolutionary Virtual Engineer
(E.V.E.).

Unlike engineering specifications, this catalog does not
define repository behavior.

Instead, it serves as the centralized inventory of official
repositories that comprise the VectorMorph ecosystem.

Repository behavior and workflows are defined by the
corresponding engineering specifications and developer
guides.

---

# Scope

This catalog includes:

- Core Repositories
- Documentation Repositories
- Platform Repositories
- Interface Repositories
- Infrastructure Repositories
- Research Repositories

Additional repositories may be added as the ecosystem
evolves.

# Repository Naming Guidelines

Repositories should:

- Represent a single logical project.
- Have a clearly defined purpose.
- Follow consistent naming conventions.
- Maintain long-term stability.
- Be documented before public release.

Each repository should remain focused on a single primary
responsibility.
