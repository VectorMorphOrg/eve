# Purpose

This document provides the authoritative catalog of
knowledge objects recognized by the Evolutionary Virtual
Engineer (E.V.E.).

Unlike engineering specifications, this catalog does not
define knowledge processing behavior.

Instead, it establishes the canonical object types that make
up E.V.E.'s engineering knowledge base.

Every documented artifact known to E.V.E. should belong to a
defined Knowledge Object.

---

# Scope

This catalog includes:

- Specifications
- Reference Catalogs
- Architecture Decision Records
- Developer Guides
- Implementation Guides
- Roadmaps
- Whitepapers
- Repository Documentation

Additional object types may be introduced as the platform
evolves.

# Object Naming Guidelines

Knowledge Objects should:

- Represent a single documented entity.
- Be uniquely identifiable.
- Have a clearly defined purpose.
- Remain stable over time.
- Be referenced consistently throughout EVE-SPEC.

Knowledge Objects define **what E.V.E. knows**.

---

# Relationship to EVE-SPEC

This catalog complements the Knowledge Layer
Specification.

Knowledge Objects form the foundation of E.V.E.'s
documentation-first architecture.

Every document referenced by E.V.E. should map to one or
more Knowledge Objects defined within this catalog.

# Summary

The Knowledge Object Catalog provides a centralized inventory
of the document types and engineering artifacts recognized by
the Evolutionary Virtual Engineer.

It establishes the common vocabulary used to organize,
reference, retrieve, and reason about project knowledge
throughout the E.V.E. platform.
